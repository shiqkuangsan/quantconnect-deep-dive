## 理解策略模板（5 分钟）

现在让我们逐行理解这个模板。每一行都很重要。

### 第 1 行：类定义

```python
class MyFirstStrategy(QCAlgorithm):
```

**这是什么：**
- `class MyFirstStrategy` — 定义一个名为 "MyFirstStrategy" 的类
- `(QCAlgorithm)` — 继承自 `QCAlgorithm`，这是 QuantConnect 提供的基类

**为什么重要：**
- `QCAlgorithm` 是 QuantConnect 的核心接口，包含了所有你需要的交易 API
- 当你继承它，你就获得了：
  - 数据订阅能力
  - 下单能力
  - 策略调度能力
  - 日志、调试能力
  - 等等

**类比：** 如果 QuantConnect 是一个游戏引擎，`QCAlgorithm` 就像是 Unity 的 `MonoBehaviour`。你写的所有代码都要继承它，才能 "活动" 在这个世界里。

### 第 2 行：初始化函数

```python
def initialize(self):
```

**这是什么：**
- `def initialize(self)` 是一个特殊的函数，在策略启动时 **恰好运行一次**
- 类似于程序的 `main()` 函数或构造函数

**你在这里做什么：**
- 设定回测时间范围
- 设定初始资金
- 订阅股票、期货等数据
- 创建指标、定时器等
- 所有一次性配置都放这里

**为什么分开：**
- `initialize()` 运行一次，在市场数据开始流入前
- `on_data()` 运行多次，每次新数据到达时
- 这样分离确保你的配置不会被意外修改

### 第 3-4 行：时间和资金配置

```python
self.set_start_date(2020, 1, 1)
self.set_cash(100000)
```

**`set_start_date(year, month, day)`：**
- 告诉回测引擎：从 2020 年 1 月 1 日开始回测
- 引擎会加载这个日期之后的所有历史数据

**`set_cash(amount)`：**
- 告诉回测引擎：你的账户初始资金是 100,000 美元
- 这不是真钱，只是模拟账户的虚拟资金

**为什么这很重要：**
- 起始日期决定了回测涵盖多少历史数据
- 初始资金影响每笔交易的头寸大小和风险
- 例如，如果改成 `set_cash(10000)`，你的交易就会缩小 10 倍

### 第 5 行：数据订阅

```python
self.add_equity("SPY")
```

**这是什么：**
- `add_equity(symbol)` 告诉 QuantConnect："我想交易 SPY 这个品种"
- **SPY** 是 SPDR S&P 500 ETF Trust，代表美国 500 大公司的指数基金

**引擎做了什么：**
1. 连接到数据源，下载 SPY 从 2020-01-01 到现在的所有历史价格数据
2. 将每条价格数据（OHLCV：开盘价、最高价、最低价、收盘价、成交量）打包成 `Bar` 对象
3. 当 `on_data()` 被调用时，以时间顺序逐条发送给你

**为什么不直接调用 API：**
- QuantConnect 为你处理了所有数据管理的复杂性
- 你不需要自己下载、清洗、去重复数据
- 不需要处理股票分割、派息等复杂调整

### 第 6-7 行：主交易逻辑

```python
def on_data(self, data):
    pass
```

**这是什么：**
- `on_data(self, data)` 是另一个特殊的函数
- 当新的市场数据到达时，这个函数就被 **自动调用一次**

**参数说明：**
- `self` — 总是指向当前策略对象
- `data` — 包含本轮新到达的数据，是一个类似字典的对象

**当回测运行时，事件流是这样的：**

```
时间 ┌─────────────────────────────────────────────┐
     │ initialize() 运行一次                         │
     │  • 设置时间、资金、订阅数据                   │
     └─────────────────────────────────────────────┘
       ↓ (2020-01-01 市场数据开始到达)
     ┌─────────────────────────────────────────────┐
     │ on_data(data) 运行一次                       │
     │  • 数据：SPY 2020-01-02 OHLCV                │
     │  • 你可以在这里决定买卖                      │
     └─────────────────────────────────────────────┘
       ↓ (下一个交易日数据到达)
     ┌─────────────────────────────────────────────┐
     │ on_data(data) 再运行一次                     │
     │  • 数据：SPY 2020-01-03 OHLCV                │
     └─────────────────────────────────────────────┘
       ↓ (不断循环...)
     [这个过程重复约 1000+ 次，直到回测结束日期]
```

**当前的 `pass` 意味着什么：**
- `pass` 是 Python 占位符，什么都不做
- 所以目前这个策略每天收到数据，但不做任何交易
- 结果：完全不赚钱，但也不赔钱

## 写一个均线交叉策略（10 分钟）

现在是真正的编码环节。我们要替换 `on_data()` 函数，写一个简单但有效的交易逻辑：**SMA 均线交叉**。

### 策略的核心逻辑

简单移动平均（SMA）交叉策略的思想非常直观：

```
价格 ↑
     │      快速线(10日SMA)
     │    ↗
   100├──────────↗────────┐
     │         ↗          │
     │       ↗ │          │↘
   95 │────────┼──────────┼───↘──
     │   ↗    │          │      ↘
     │ ↗      │          │       ↘
   90├────────│──────────┼────────┘
     │ 慢速线(30日SMA)    │    死亡叉(卖出信号)
     │        │          ↓
     │        │     交叉点
     │        ↑
     │    黄金叉(买入信号)
     └─────────────────────────→ 时间
```

**规则很简单：**
1. 每天计算最近 10 天 SPY 的平均价格（快速线）
2. 每天计算最近 30 天 SPY 的平均价格（慢速线）
3. 当快速线从下方穿过慢速线时 → **买入** 信号（黄金叉）
4. 当快速线从上方穿过慢速线时 → **卖出** 信号（死亡叉）

### 完整策略代码

现在请 **完全替换** 编辑器中的代码。清空所有现有代码，复制粘贴以下内容：

```python
from AlgorithmImports import *

class MyFirstStrategy(QCAlgorithm):

    def initialize(self):
        # ============ 策略参数配置（一次性） ============
        # 设置回测的开始和结束日期
        # 为什么从 2019 开始？因为我们需要 30 天数据来预热均线指标
        self.set_start_date(2019, 1, 1)

        # 设置回测结束日期（不设置则默认到最新）
        # self.set_end_date(2024, 12, 31)

        # 设置初始现金
        # 单位是美元，模拟账户
        self.set_cash(100000)

        # 添加 SPY（美国 500 大公司指数 ETF）
        # 这会告诉 LEAN 引擎下载并推送 SPY 的日线数据给我们
        self.add_equity("SPY", Resolution.Daily)

        # ============ 创建均线指标 ============
        # 快速均线：过去 10 天的简单移动平均
        # SMA 会自动订阅到 SPY，每天更新一次值
        # self.fast_sma 是一个 IndicatorBase 对象，包含历史值序列
        self.fast_sma = self.SMA("SPY", 10)

        # 慢速均线：过去 30 天的简单移动平均
        # 同样自动更新
        self.slow_sma = self.SMA("SPY", 30)

        # ============ 交易状态记录 ============
        # 用一个布尔值记录我们当前是否持仓
        # True 表示当前持有 SPY，False 表示没有持有
        # 我们用这个来避免重复买入或卖出
        self.is_holding = False

        # ============ 日志设置 ============
        # 设置日志输出的详细级别
        # 这样我们能在控制台看到策略的每一步动作
        self.debug(f"初始化完成。初始资金：$100,000")


    def on_data(self, data):
        # ============ 每个交易日都执行一次 ============
        # 当新的市场数据（OHLCV）到达时，这个函数被调用

        # 安全检查 1：检查数据是否就绪
        # 如果 SPY 今天没有新的价格数据，就跳过这一轮
        # 这在 LEAN 回测引擎中很少发生，但在实盘中可能出现（停盘等）
        if "SPY" not in data:
            return

        # 安全检查 2：检查均线指标是否预热完成
        # 指标需要足够的历史数据才能给出有意义的值
        # 快速均线需要 10 天，慢速均线需要 30 天
        # is_ready 属性在有足够数据后变为 True
        if not self.fast_sma.IsReady or not self.slow_sma.IsReady:
            # 日志：告诉我们当前还在预热阶段
            # 这通常是回测的前 30 天
            self.debug("指标预热中，等待...")
            return

        # ============ 获取当前数据 ============
        # 获取今天的收盘价
        # data["SPY"] 是今天的价格 Bar 对象
        # data["SPY"].Close 是收盘价（浮点数）
        price = data["SPY"].Close

        # 获取快速均线和慢速均线的当前值
        # .Current.Value 得到均线的最新值（浮点数）
        fast = self.fast_sma.Current.Value
        slow = self.slow_sma.Current.Value

        # ============ 交易逻辑：黄金叉买入 ============
        # 条件 1：当前没有持仓（avoid 重复买入）
        # 条件 2：快速均线 > 慢速均线（价格向上穿过）
        # 条件 3：确保快速线真的在慢速线上方（避免噪声误触）
        if not self.is_holding and fast > slow:
            # 购买 100 股 SPY
            # self.buy() 是核心交易函数
            # 参数：
            #   symbol="SPY" — 买什么
            #   quantity=100 — 买多少股
            # 这个订单会立即提交到模拟交易系统
            self.buy("SPY", 100)

            # 更新持仓状态
            self.is_holding = True

            # 日志记录
            # debug() 的消息会出现在下方的控制台
            # 这样我们可以在回测结束后回顾整个交易历史
            self.debug(f"[黄金叉] 快速均线 ({fast:.2f}) > 慢速均线 ({slow:.2f})，买入 100 股 @ ${price:.2f}")

        # ============ 交易逻辑：死亡叉卖出 ============
        # 条件 1：当前持有 SPY（avoid 重复卖出）
        # 条件 2：快速均线 < 慢速均线（价格向下穿过）
        elif self.is_holding and fast < slow:
            # 卖出全部持仓
            # self.liquidate() 是 "清空" 函数，卖出该品种的所有头寸
            self.liquidate("SPY")

            # 更新持仓状态
            self.is_holding = False

            # 日志记录
            self.debug(f"[死亡叉] 快速均线 ({fast:.2f}) < 慢速均线 ({slow:.2f})，卖出全部 @ ${price:.2f}")


# ============ 脚注说明 ============
# 为什么不直接比较前一天和今天的快速均线？
# 因为均线是平滑的，我们想要的是 "方向改变"，而不是 "具体值改变"。
# 由于均线是连续变化的，直接比较当前值和历史值容易被噪声迷惑。
# 在实际应用中，专业的量化交易者会使用更复杂的穿叉检测算法，
# 例如检查前一根 bar 是否 fast < slow，当前 bar 是 fast > slow。
# 但对于这个简单的演示，当前逻辑已足够演示策略框架的工作流程。
```

### 代码分解

上面的代码看起来很长，但逻辑简单。让我们按段理解：

**第 1 行：导入**
```python
from AlgorithmImports import *
```
- 导入 QuantConnect 提供的所有基础函数和类
- `*` 表示导入所有，这样 `SMA`、`Resolution` 等都可以直接使用

**`initialize()` 函数：一次性设置**
```python
self.set_start_date(2019, 1, 1)          # 回测从 2019 年开始
self.set_cash(100000)                    # 初始资金 10 万美元
self.add_equity("SPY", Resolution.Daily) # 订阅 SPY 日线数据
self.fast_sma = self.SMA("SPY", 10)      # 创建 10 日均线
self.slow_sma = self.SMA("SPY", 30)      # 创建 30 日均线
self.is_holding = False                  # 初始没有持仓
```

**`on_data()` 函数：每日交易逻辑**
```python
# 检查数据和指标是否就绪
if "SPY" not in data or not self.fast_sma.IsReady:
    return

# 获取当前价格和均线值
price = data["SPY"].Close
fast = self.fast_sma.Current.Value
slow = self.slow_sma.Current.Value

# 黄金叉：快线从下穿过慢线 → 买入
if not self.is_holding and fast > slow:
    self.buy("SPY", 100)
    self.is_holding = True

# 死亡叉：快线从上穿过慢线 → 卖出
elif self.is_holding and fast < slow:
    self.liquidate("SPY")
    self.is_holding = False
```

### 重点笔记

| 概念 | 含义 | 代码 |
|------|------|------|
| **订阅** | 告诉引擎你关注某个品种 | `self.add_equity("SPY")` |
| **指标** | 自动计算的技术指标（均线、RSI 等）| `self.SMA("SPY", 10)` |
| **预热** | 指标需要足够历史数据才能计算 | `self.fast_sma.IsReady` |
| **黄金叉** | 快线穿过慢线向上 → 买入信号 | `if fast > slow` |
| **死亡叉** | 快线穿过慢线向下 → 卖出信号 | `elif fast < slow` |
| **持仓记录** | 用布尔值跟踪当前是否持有头寸 | `self.is_holding` |
| **下单** | 提交购买或卖出命令 | `self.buy()` / `self.liquidate()` |
| **日志** | 记录关键决策点，便于调试 | `self.debug(message)` |

### 关键指标详解

当你看到报告时，会看到一堆数字和百分比。以下是最重要的几个，每个都有具体解释：

#### 1. **总收益率（Total Return）**

```
总收益率: +45.23%
```

**含义：**
- 从开始到结束，你的账户增长了 45%
- 用公式：(最终资金 - 初始资金) / 初始资金
- 例子：从 $100,000 变成 $145,230

**为什么重要：**
- 这是最直观的性能指标
- 但它没有考虑到时间因素（10 年赚 45% vs 1 年赚 45% 是两回事）

**什么是好的总收益率：**
- 年化 10% 以上是不错的（因为 S&P 500 长期平均年化 10%）
- 年化 20% 以上是优秀的
- 但这取决于你的风险承受度

#### 2. **年化收益率（Annualized Return）**

```
年化收益率: 8.23%
```

**含义：**
- 如果你每年都以这个速率赚钱，总收益率会是多少
- 公式较复杂，但核心思想是：平均每年赚多少百分比

**例子：**
- 如果回测 5 年，总收益率是 45%，那年化收益率约 8%
- 如果回测 1 年，总收益率是 15%，那年化收益率约 15%

**为什么这很重要：**
- 不同长度的回测结果难以比较，年化收益率解决这个问题
- 用年化收益率可以公平地比较不同时间段的策略

**什么是好的年化收益率：**
- 8-10%：和 S&P 500 差不多，说明你的策略没有超额收益
- 15-20%：很好，说明你的策略有明显优势
- 30%+：非常好，但要警惕过优化或数据泄露

#### 3. **最大回撤（Max Drawdown）**

```
最大回撤: -18.5%
```

**含义：**
- 从账户的历史高点到最低点，最大跌幅是多少
- 换个说法：最坏的时刻，你赔了多少

**例子：**
- 假设你的账户曾经达到过 $120,000 的高点
- 后来跌到 $98,000
- 回撤 = ($98,000 - $120,000) / $120,000 = -18.3%

**为什么这很重要：**
- 这是风险指标，反映了策略在最坏情况下的表现
- 如果最大回撤是 -50%，意味着即使策略长期赚钱，你也可能在某些时刻亏损一半
- 心理上能承受吗？

**什么是可接受的最大回撤：**
- 20-30%：可以接受，在股市中很常见
- 30-50%：高风险，需要很强的心理素质
- 50%+：极高风险，通常不建议实盘交易

**图表形象：**
```
账户价值
  $130k ┌─ 历史高点
        │  ╱╲
  $120k│ ╱  ╲
        │╱    ╲╲
  $110k├      ╲ ╲
        │       ╲ ╲
  $100k│        ╲ └─ 最大回撤点（-18.5%）
        │         ╲  ╱
   $98k│          ╲╱
        └──────────────→
```

#### 4. **夏普比率（Sharpe Ratio）**

```
夏普比率: 1.24
```

**含义：**
- 衡量 "单位风险下的收益"
- 公式：(年化收益率 - 无风险利率) / 年化波动率
- 简单理解：如果两个策略收益相同，风险越小的夏普比率越高

**例子：**
- 策略 A：赚 20%，波动率 15%，夏普比率 = (20% - 2%) / 15% = 1.2
- 策略 B：赚 20%，波动率 30%，夏普比率 = (20% - 2%) / 30% = 0.6
- 策略 A 更优，因为风险更小

**什么是好的夏普比率：**
- 0.5-1.0：不错
- 1.0-2.0：很好
- 2.0+：优秀

**为什么这很重要：**
- 专业投资者看的是夏普比率，而不仅是收益率
- 一个夏普比率 0.5 的策略如果加杠杆，夏普比率不会提高
- 但一个夏普比率 2.0 的策略加杠杆，可以获得更好的风险调整后收益

#### 5. **胜率（Win Rate）**

```
胜率: 52.3%
```

**含义：**
- 在所有交易中，有多少笔是盈利的
- 公式：盈利交易数 / 总交易数

**例子：**
- 你做了 100 次交易，其中 52 次赚钱，48 次亏钱
- 胜率 = 52%

**什么是好的胜率：**
- 50%：和抛硬币一样，没有优势
- 55%：不错，说明策略有方向性优势
- 60%+：很好
- 但注意：胜率不等于赚钱！因为赚的和赔的可能不对称

**例子：胜率高但亏钱的情况：**
- 你做了 10 次交易
- 9 次赚 $100，1 次赔 $1000
- 胜率 = 90%，但总 P&L = $900 - $1000 = -$100
- 所以胜率只是指标之一

#### 6. **盈亏比（Profit Factor）或 平均盈亏（Avg Win/Avg Loss）**

```
平均盈利: $450
平均亏损: $300
盈亏比: 1.5
```

**含义：**
- 当你赚钱时，平均赚多少
- 当你亏钱时，平均亏多少
- 盈亏比 = 平均盈利 / 平均亏损

**例子：**
- 50 次交易赚钱，平均 $500
- 50 次交易亏钱，平均 $400
- 盈亏比 = $500 / $400 = 1.25

**什么是好的盈亏比：**
- 1.0：平均赚和亏一样多，和胜率组合才能判断好坏
- 1.5-2.0：不错，说明赚的比亏得多
- 2.0+：很好
- 如果胜率 50% + 盈亏比 2.0，那么每次交易期望收益 = 50% × $2 - 50% × $1 = +$0.5（正期望）

**胜率与盈亏比的组合：**

| 胜率 | 盈亏比 | 评价 |
|------|---------|------|
| 60% | 0.8 | 经常赚钱，但赚得少，风险 |
| 40% | 3.0 | 经常亏钱，但赚时大赚，可能可行 |
| 55% | 1.5 | 理想组合，稳定盈利 |
| 45% | 2.5 | 不常赚钱，但赚时很多，需要承受心理压力 |

### 优化 2：加入止损逻辑

目前的策略只在均线交叉时卖出，可能会承受很大的回撤。我们加入一个简单的止损机制：持仓后，如果价格下跌超过 5%，立即卖出。

**修改 `on_data()` 函数中的买入部分：**

替换这段：
```python
if not self.is_holding and fast > slow:
    self.buy("SPY", 100)
    self.is_holding = True
    self.debug(f"[黄金叉] 快速均线 ({fast:.2f}) > 慢速均线 ({slow:.2f})，买入 100 股 @ ${price:.2f}")
```

为这段：
```python
if not self.is_holding and fast > slow:
    self.buy("SPY", 100)
    self.is_holding = True
    # 记录买入价格，用于止损计算
    self.entry_price = price
    self.debug(f"[黄金叉] 快速均线 ({fast:.2f}) > 慢速均线 ({slow:.2f})，买入 100 股 @ ${price:.2f}")
```

然后在 `initialize()` 中加一行：
```python
self.entry_price = None  # 买入价格，用于止损
```

最后，修改卖出逻辑：
```python
# 检查止损条件：持仓后，价格下跌超过 5%
elif self.is_holding and self.entry_price is not None:
    loss_pct = (price - self.entry_price) / self.entry_price

    # 止损条件：亏损超过 5%
    if loss_pct < -0.05:
        self.liquidate("SPY")
        self.is_holding = False
        self.entry_price = None
        self.debug(f"[止损] 亏损 {loss_pct*100:.2f}%，清仓 @ ${price:.2f}")

    # 死亡叉卖出条件（保留原来的逻辑）
    elif fast < slow:
        self.liquidate("SPY")
        self.is_holding = False
        self.entry_price = None
        self.debug(f"[死亡叉] 快速均线 ({fast:.2f}) < 慢速均线 ({slow:.2f})，卖出全部 @ ${price:.2f}")
```

**保存并重新回测。**

**对比结果：**

| 指标 | 无止损 | 有 5% 止损 |
|------|---------|----------|
| 总收益率 | 48.15% | 42.80% |
| 最大回撤 | -16.2% | -5.8% |
| 交易次数 | 38 | 58 |

**分析：**
- 加止损后，最大回撤显著减少（从 -16.2% 到 -5.8%）
- 但总收益率下降了（因为有些好的趋势被止损打断了）
- 交易次数增加了（止损后会重新进场）
- 权衡：风险大幅降低，代价是一些收益

**你应该选哪个？**
- 如果你睡眠质量重要，选有止损的版本
- 如果你追求最大收益且心理素质强，选无止损的版本

### 做笔记：参数优化的正确姿态

这里有一个重要警告：**不要过度优化**。

常见的错误：
```python
# 坏的做法：试遍所有参数
for fast_period in range(5, 50):
    for slow_period in range(fast_period, 200):
        for stop_loss_pct in [0.01, 0.03, 0.05, 0.10]:
            # 回测组合...
            # 找到最好的参数
            # 然后用这个参数实盘...
            # 结果：表现远差于回测（过优化）
```

**正确的做法：**
1. 选择 **有理论支撑** 的参数（例如，20/50 均线是经典的趋势跟踪组合）
2. 在 **不同资产和时间段** 上验证
3. 接受回测和实盘的差异（通常实盘差 30-50%）
4. 保持参数相对稳定，通过 **增加头寸、多品种、多策略** 来提升收益

## 常见问题 FAQ

### Q1：回测显示 0 笔交易，什么意思？

**症状：**
```
Total Trades: 0
Total Return: 0%
```

**常见原因：**

1. **指标没有准备好（IsReady = False）**
   - 检查你的代码是否有 `if not self.fast_sma.IsReady: return`
   - 如果没有，指标值会是 `None` 或 `0`，导致逻辑永不触发

2. **交易信号的条件永不满足**
   - 打印中间值看看：
   ```python
   self.debug(f"Price: {price}, Fast: {fast}, Slow: {slow}")
   ```
   - 如果 Fast 和 Slow 总是相等或很接近，说明逻辑有问题

3. **品种没有数据**
   - 检查 `self.add_equity("SPY")` 是否写对了
   - 试试改成 `self.add_equity("AAPL")` 看是否有交易

### Q2：错误 "'NoneType' has no attribute 'Close'"，怎么办？

**症状：**
```
Error: 'NoneType' object has no attribute 'Close'
  at line: price = data["SPY"].Close
```

**原因：**
- `data["SPY"]` 是 `None`，说明今天没有 SPY 的数据
- 可能是：
  - 数据在某个日期缺失（罕见）
  - 品种没有订阅
  - 时间范围设错了

**解决：**
```python
# 永远检查数据是否存在
if "SPY" not in data:
    return

# 然后再访问
price = data["SPY"].Close
```

### Q3：回测结果看起来太好了（年化 100%+），是不是有问题？

**是的，很可能是。** 这叫 **过优化** 或 **过拟合**。

常见原因：

1. **前向偏差（Look-ahead bias）**
   - 你的代码用了 "未来" 的数据
   - 例如：
   ```python
   # 错误：用了 tomorrow 的价格，但今天不可能知道
   tomorrow_price = ...
   if price > tomorrow_price:
       self.liquidate()
   ```

2. **生存者偏差（Survivorship bias）**
   - 只用了还活着的公司的数据
   - 例如，某家公司 2010 年破产了，你的数据里没有它
   - 所以你不会在它破产前被清出来

3. **参数过度优化**
   - 参数是为了历史数据专门调的
   - 例如 "这些参数在 2020-2024 最优"，但 2019 和 2025 会很差

**检查过优化的方法：**

在不同时间段回测（Walk-forward analysis）：
- 第 1 周期：2019-2021 回测，得出最优参数
- 第 2 周期：用这个参数，在 2021-2023 回测（没有优化过的新数据）
- 如果第 2 周期的结果也不错，说明参数是健壮的

### Q4：想在不同时间范围测试，怎么改？

**改这两行：**
```python
# 改开始日期
self.set_start_date(2019, 1, 1)

# 改结束日期（不改的话默认到最新）
self.set_end_date(2024, 12, 31)
```

**试试这些组合：**
```python
# 只回测最近 1 年
self.set_start_date(2024, 1, 1)
self.set_end_date(2024, 12, 31)

# 回测 10 年
self.set_start_date(2014, 1, 1)
# 不写 set_end_date，默认到最新

# 回测 2008 年金融危机
self.set_start_date(2008, 1, 1)
self.set_end_date(2009, 12, 31)
```

**观察：** 策略在不同时期的表现会不同。好的策略应该在多个时期都有正收益。

### Q5：实盘交易和回测差很大，为什么？

**常见的 10 个原因：**

| 原因 | 回测 | 实盘 |
|------|------|------|
| 滑点（Slippage） | 假设按收盘价成交 | 实际可能差 0.01-0.1% |
| 佣金费用 | 默认 0 或很低 | 可能 0.1-1% 每单 |
| 流动性 | 假设总能成交 | 低流动性品种可能成交困难 |
| 数据延迟 | 回测用的是官方数据 | 实盘可能有 100ms 延迟 |
| 市场情绪变化 | 过去的数据 | 市场可能有新的相关性 |
| 看穿 | 不会发生 | 某些交易可能被平台拒绝 |
| 极端行情 | 正常运行 | 熔断、跌停可能触发 |
| 策略过优化 | 因为用的就是这段历史 | 新数据不适配 |
| 执行差错 | 代码完美运行 | 网络问题、系统故障 |
| 本金规模 | 可以瞬间执行大单 | 大单会推动价格 |

**实测数据：** 一个年化 15% 的回测策略，实盘通常年化 8-12%（打个五折到七折）。如果你看到年化 100% 的回测，实盘可能是 0% 甚至负数。
